# P2 — A real distributed interaction-net runtime (status)

`p2.py` takes the project from in-process *simulation* (`inet.py`) to a real distributed runtime:
**independent workers that share nothing but messages**, running the spec's boundary-owner protocol,
with **Safra's distributed termination detection** as the control plane. It is verified against the
sequential reducer in `inet.py` as the oracle.

## What's actually built

- **Separate per-worker heaps.** `partition()` splits one net into N `Worker`s; every cross-node wire
  becomes two **boundary stubs** joined by a global `wire_id`. After partition, no port object is
  shared across workers — workers exchange only plain-data messages (`export` descriptors + the token).
- **Boundary-owner protocol.** Each wire has a deterministic owner (`owner_min` = lower node id). The
  non-owner ships its agent's *header* (kind + aux `wire_id`s, never the subtree) to the owner; the
  owner reconstructs, splices internal wires, and performs the now-local rewrite. (`Worker.export` /
  `Worker.inject`.)
- **Safra termination detection.** A token ring with per-worker counters (`c = sends − receives`) and
  colors (blacken on receiving work). A white token returning white to node 0 with `count + c_0 == 0`
  means the whole computation is globally finished. (`run_threaded`.)

## What's validated (run `python3 p2.py`)

1. **Protocol over separate heaps** — 40/40 random partitions/schedules reduce to the *same* normal
   form as the sequential oracle, using real export/inject messages between disjoint heaps.
2. **Concurrent runtime + termination** — 20/20 runs of real threads (shared-nothing, message-passing
   only): Safra **detected** global termination every time (no early stop, no hang), and every stitched
   result matched the oracle.

## What building it taught us (folded into SPEC §4.4)

- **Per-channel FIFO is a real requirement, not an optimization.** An exported agent mints new boundary
  wires; a child export can reference a wire a parent export just created on the owner. With arbitrary
  global message reordering the owner receives the child before the wire exists and crashes. The fix is
  **per-directed-pair FIFO delivery** — which TCP and BEAM give for free (so it costs no consensus and
  doesn't weaken the coordination-free claim), but it *is* an assumption worth stating. The first test
  build failed exactly here; FIFO delivery made it green.
- This is the "build reveals a spec gap" loop working as intended: the theory said "monotone messages,
  reorder-robust"; the implementation showed the robustness is *per-channel*, not global.

## Honest scope (what this is NOT yet)

- **Threads, not OS processes or BEAM.** Workers are shared-nothing threads with `queue.Queue` channels.
  This faithfully exercises the *algorithm* (concurrent reduction, message passing, FIFO channels,
  Safra) — the BEAM/Zig port is a transport swap, not an algorithm change — but it is not multi-machine.
- **One toy net, `owner_min`.** The workload is the depth-3 duplication net; `owner_min` funnels all
  boundary work to node 0 (correct but lopsided). A hash-based owner would balance it.
- **No performance, scale, or fault recovery yet.** This proves *correctness + termination*, which is
  the P2 milestone. Perf/scale/recovery are P3–P4.

## Next

- **P2.1** swap threads for OS processes (then BEAM nodes): identical protocol + Safra over a real
  network; `owner = hash(wire_id)` for balance; measure the local-vs-remote interaction ratio — the
  number that decides whether distribution pays.
- **P3** locality-aware partitioner (keep interacting agents co-located; don't distribute sequential
  fragments).
- **P4** fault recovery via deterministic recomputation of a lost worker (sharing-aware lineage).
